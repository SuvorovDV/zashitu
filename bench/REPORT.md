## Failure modes / observations

1. **source_ref фичи много используют формат «Раздел "..."», а не «с. N»** — потому что в seeded speech.md не было разметки страниц (это речь без привязки к номерам страниц исходника). На реальных заказах с PDF-источником Claude должен бы чаще попадать на «с. 47». Надо провалидировать на живом прод-заказе с реальным PDF.

2. **Два слайда (section + quote) корректно без source_ref.** Это штатное поведение, не регрессия.

3. **Все named entities из after-дека присутствуют в seed-speech.md** — проверил визуально. NER-галлюцинаций не было. Но валидатор `validate_named_entities` из agent 4 НЕ реализован — это отложено до следующей итерации (будет нужен, как только заказы пойдут на реальных PDF).

4. **Typography / color / accents остались baseline-уровня.** Это Path A, следующая неделя.

## Риски перед деплоем

1. **Прод-пайплайн под real-PDF orders** — бенч проверен только на seeded speech. Возможен drift на реальных PDF. Mitigation: monitored first 5 prod orders после деплоя, rollback если `has_source_ref coverage < 80%`.

2. **Breaking change в JSON schema?** — Добавлено поле `source_ref` в Pydantic. По умолчанию `Optional[str] = None`. Старые orders без source_ref не ломаются. Fallback сработает, проставит «источник: загруженная работа».

3. **pptxgen.js** content-area shrunk с 5.25→5.0 inch. Возможна регрессия на слайдах с длинным bullet-текстом (обрезка). Не наблюдал на smb_digital, но надо проверить при ревью.
