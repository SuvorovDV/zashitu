# Roadmap продукта

> Версия 3. Фокус: TG-бот (не Mini App), без написания дипломных работ.

---

## 1. Что это

AI-ассистент для студента: от темы до успешной защиты. Не «генератор презентаций», а инструмент подготовки к выступлению.

**Рабочее название:** ZaShitu / Акад — финализировать позже.

**Главное отличие от конкурентов** (Gamma, Tome, российские аналоги): они делают красивую презентацию с выдуманными цифрами. Мы делаем презентацию по реальной работе студента + доклад + Q&A + тренажёр. Студент идёт на защиту подготовленным, а не с красивыми слайдами и пустыми ответами.

**Что НЕ делаем принципиально:**
- Написание дипломных и курсовых работ — антиплагиат, юридика, масштаб
- Диссертации PhD и научные статьи
- «Чат с ИИ для учёбы» — это ChatGPT
- Социальную сеть студентов

## 2. Состояния входа пользователя

```
1. Есть готовая работа (курсовая / ВКР / реферат)
   → Презентация + доклад + Q&A + тренажёр

2. Есть только тема
   → Презентация с нуля + доклад к ней

3. Есть методичка кафедры / паспорт проекта
   → Заполнение по теме + презентация
```

## 3. Модули продукта

### 3.1. Презентация (MVP)
Генерация .pptx. Три режима:
- `no_template` — с нуля по теме (от школьного доклада до курсовой)
- `source_grounded` — по загруженной работе PDF/DOCX, каждый слайд привязан к странице источника
- `structure_template` — заполнение методички / паспорта проекта

Масштаб: 8–10 слайдов (доклад) до 25–30 (академическая по ГОСТу).

### 3.2. Доклад (этап 2)
Текст выступления с хронометражем по слайдам. Выход — .docx со структурой «слайд N (X мин) — что говорить», тайминг, паузы.

### 3.3. Q&A (этап 2)
10–15 предсказанных вопросов комиссии с ответами. В режиме `source_grounded` — ссылки на конкретные страницы работы.

### 3.4. Тренажёр защиты (этап 3, killer feature)
Интерактивный диалог: Claude задаёт вопросы — студент отвечает — Claude даёт обратную связь. Требует веб-интерфейс. Ни у одного конкурента нет.

### 3.5. Код (этап 4)
Простой артефакт для показа на защите: веб-страничка, Excel/VBA, Python-скрипт, SQL. 50–200 строк, не продакшен.

## 4. Платформы

| Платформа | Статус | Назначение |
|---|---|---|
| **Telegram-бот** | MVP | Основной канал: диалог, сбор данных, приём файлов, оплата Stars, выдача .pptx |
| **Веб-версия** | Этап 3 | Тренажёр защиты, длинные сессии, подписки |
| **PWA** | Этап 3 | Мобильный доступ к веб-версии |

Нативные iOS/Android — не делаем.

**Важно про TG-бот vs Mini App:** делаем именно бота (python-telegram-bot / aiogram). Никакого WebApp/Mini App. Бот ведёт пользователя через диалог пошагово: собирает поля, принимает файл, запускает генерацию, отдаёт файл. Оплата — через Telegram Stars (`send_invoice`).

## 7. Открытые вопросы

**Юридика:** оферта «справочный материал, ответственность на пользователе». Чистая зона — мы не пишем работу.

**TG Stars:** 30% комиссия Telegram — закладывай в цены. Recurring payments через Stars технически есть, но менее стабильны — на MVP делаем только разовые.

**Себестоимость:** презентация 20 слайдов по 40-страничному источнику ≈ $0.15–0.25 на Sonnet, ≈ $0.40–0.70 на Opus. Маржа комфортная на стандарте и премиуме.

## 8. Что не делаем

- Дипломные и курсовые работы (только презентация по ним)
- Диссертации PhD и статьи в журналы
- TG Mini App / WebApp — только классический бот
- Нативные мобильные приложения
- Сложное ПО в модуле «Код»
- Социальную сеть и форумы
